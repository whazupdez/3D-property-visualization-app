3. Functional Requirements

Section 1: Overview (Landing)

Left Panel (60%)

  • Hero image/video
  • Property name and tagline
  • Key visual highlights

Right Panel (40%)

  • Price and key specifications
  • Property description (brief)
  • Key features (5-6 points)
  • Navigation to other sections

Section 2: Location & Amenities

Left Side (50%)

  • Interactive map (large, TV-friendly)
  • Property location marker
  • Zoom controls

Right Side (50%)

  • Address information
  • Nearby amenities with distances
  • Transportation links
  • Neighborhood highlights

Section 3: Units & Floor Plans (3-Step Process)

Step 1: Unit Type Selection

Large cards with unit types (1BR, 2BR, 3BR, Penthouse) showing key specs and pricing

Step 2: Floor Plan Details

Split screen: Large floor plan (60%) + Unit specifications (40%)

Step 3: Unit Gallery

Large main image (70%) + Thumbnail strip (30%) with category filters

Section 4: Virtual Tours (Tabbed Interface)

Tab 1: 3D Exterior

  • Full screen 3D building model
  • Rotate, zoom, pan controls
  • Day/night toggle
  • Reset view button
  • Screenshot capture

Tab 2: 360° Interior

  • AIHouse.com embedded tour
  • Room navigation
  • Fullscreen mode
  • Mobile touch controls
  • Fallback to static images

AIHouse.com Integration

URL: https://www.aihouse.com/720/QQO202507310000000000000925860600 Method: Responsive iframe with postMessage communication Fallback: Static panoramic images if tour fails to load

  • Office hours and location

Right Panel (60%)
